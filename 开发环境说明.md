# 开发环境说明

## 当前目录结构
- `upstream/ant-design-pro`
  - 官方上游基线，只读参考
- `upstream/ant-design-mobile`
  - 官方上游基线，只读参考
- `apps/admin-web`
  - 基于 `ant-design-pro` 的总控端/老师端 Demo
- `apps/parent-h5`
  - 基于 `ant-design-mobile` 的家长端 Demo

## 本地隔离策略
- 没有执行任何全局 `npm install -g`
- 没有安装全局 `pnpm`
- 当前项目根目录下新增了 `.npmrc`
  - `cache=.npm-cache`
  - 所有 npm 依赖缓存优先写入当前目录

## 已使用的本机能力
- 复用了本机已有的 `git`
- 复用了本机已有的 `node`
- 通过 `npm.cmd` 规避了 PowerShell 执行策略对 `npm.ps1` 的限制

## 对本机环境的影响说明
### 没有影响全局配置的部分
- 没有改系统环境变量
- 没有改全局 npm 配置
- 没有装全局包
- 没有写入非当前目录的项目文件

### 需要知道但影响较小的部分
- `npm` 运行过程中仍可能使用用户级网络/证书能力
- `npx create-vite` 会通过 npm 拉取模板包，但缓存已定向到当前目录的 `.npm-cache`

## 当前可用命令
### 总控端 / 老师端
```powershell
cd f:\Workspace_VS\CamAI-Sec\apps\admin-web
npm.cmd run dev
```

说明：
- 管理端当前使用本地 mock 数据支撑登录和演示流程
- `dev` 脚本已经保留 mock，不要改成 `MOCK=none`

### 家长端
```powershell

npm.cmd run dev
```

### 构建验证
