# 项目方向文档：智慧校园 AI 安防育人平台

## 1. 项目一句话定义
做一个面向校园场景的 AI 安防与育人协同平台，先用“可演示、可闭环、可扩展”的方式验证核心能力，而不是一开始追求全量上线。

## 2. 项目目标
### 业务目标
- 用 AI 视觉替代部分传统物理防护。
- 用主动预警替代事后查录像。
- 用统一平台打通校园、家长、老师、管理侧的信息流。

### 第一阶段目标
- 先做一个可展示的一期 demo。
- Demo 重点验证三类能力：
  - 人脸门禁 / 人员通行管控
  - 校园智能巡检 / 风险告警
  - 学生群体情绪感知 / 风险提示

## 3. 需求理解
文档表达的是一个“平台 + 两条主线”的方案：

- 安防主线：7x24 小时巡检、入侵/攀爬/抛物/溺水等风险识别、门禁和轨迹管控、SOS 联动。
- 育人主线：群体情绪感知、心理风险辅助研判、家校协同。

一期是刚需能力，二期和三期属于增强和扩展能力，不应进入当前 demo 的主实现范围。

## 4. Demo 范围建议
### 必做
- 首页总览大屏
- 门禁通行记录与告警卡片
- 校园巡检告警列表
- 群体情绪趋势看板
- 统一告警中心

### 可演示但可简化
- 模拟视频流接入
- 模拟 AI 告警触发
- 轨迹热力图
- 事件闭环流转

### 暂不做
- 真正的硬件接入
- 真正的人脸识别模型训练
- 无人机联动
- 北斗定位电子卡
- 公安系统互通

## 5. 产品形态建议
第一版 demo 建议做成一个 Web 管理后台，理由很直接：

- 演示成本低。
- 适合展示大屏、列表、告警、流程闭环。
- 后续容易接入真实算法服务和设备数据。

## 6. 第一版核心页面
1. 总览首页
   - 今日告警数
   - 在线摄像头数
   - 风险等级统计
   - 最近事件流

2. 安防巡检页
   - 摄像头/点位列表
   - 风险事件时间线
   - 告警详情

3. 门禁管理页
   - 人员通行记录
   - 黑名单/陌生人告警
   - 异常轨迹提示

4. 情绪研判页
   - 群体情绪趋势
   - 班级/年级维度统计
   - 人工复核标记

5. 告警中心
   - 待处理 / 处理中 / 已关闭
   - 处理人、处理时间、备注

## 7. 数据与合规方向
这个项目最关键的不是“炫技”，而是“可控、可审计、可解释”。

- 人脸/情绪/定位都属于敏感数据，demo 里尽量只展示脱敏数据。
- 情绪感知建议只做群体级别，不做个人标签。
- 告警图片和事件记录要有留痕，但 demo 中可用模拟数据替代真实采集。
- 所有 AI 结论都应支持人工复核。

## 8. MVP 实现建议
### 数据源
- 先用模拟事件流。
- 再预留真实 API 接口。

### 核心能力
- 事件生成
- 事件展示
- 事件流转
- 统计聚合
- 告警提醒

### 技术优先级
1. 先把页面和信息架构搭起来。
2. 再把事件模型和假数据跑通。
3. 最后接入真实推理接口。

## 9. 成功标准
demo 第一版能做到：
- 看得懂：一眼能理解校园安全和育人两条主线。
- 演得出来：能模拟风险事件并在界面上闭环展示。
- 扩得出去：后续能平滑接真实设备和算法接口。

## 10. 下一步建议
1. 确认 demo 的技术栈。
2. 确认先做 Web 管理后台还是大屏展示页。
3. 定义 5 到 8 个核心事件类型。
4. 定义事件数据结构。
5. 开始做第一版页面和假数据。

## 11. 原始资料结论
文档里明确的一期重点是：
- 人脸识别门禁 / 人员轨迹管控
- 校园智能安全巡检
- 学生情绪感知

因此 demo 应围绕这三块展开，其他能力作为路线图展示即可。
