# ADR-001: 第二阶段 Demo 继续采用双端前端 + 统一事件模型

## Status
Accepted

## Date
2026-06-19

## Context
当前项目已经完成一个可演示的校园平台壳体 Demo，但后续需要继续补齐一期业务表达，包括门禁、巡检、情绪研判和统一告警。

如果继续按页面堆叠方式推进，后续会频繁出现：
- 页面与页面之间数据断裂
- 目录结构越来越乱
- 老师端、总控端、家长端各自维护一套假数据
- 后续接真实接口时大量重构

## Decision
第二阶段继续采用：

1. 双端前端结构
   - `apps/admin-web` 承担总控端和老师端
   - `apps/parent-h5` 承担家长端

2. 统一事件模型驱动
   - 三端共享统一实体和统一事件语义
   - 页面仅是不同角色对同一数据的不同视图

3. 业务域分层
   - `overview`
   - `access-control`
   - `patrol`
   - `emotion`
   - `alert-center`
   - `teacher-workbench`
   - `parent-communication`

## Alternatives Considered

### 方案 A：继续按页面零散推进
- 优点：短期看起来最快
- 缺点：后续结构会迅速失控
- 结论：拒绝

### 方案 B：把老师端拆成独立前端
- 优点：角色隔离更彻底
- 缺点：当前阶段会引入额外维护成本
- 结论：暂不采用

### 方案 C：先做重后端，再接前端
- 优点：未来可能更接近真实系统
- 缺点：当前目标仍是高完成度 Demo，不适合引入复杂度
- 结论：拒绝

## Consequences
- 第二阶段会先投入一部分时间稳定结构，而不是立刻补很多页面。
- 后续补功能时，目录、数据和路由改动会更小。
- 当未来接真实模型或真实接口时，替换边界会更清晰。
