# 甲方演示脚本

## 1. 演示目标
在 3 到 5 分钟内，让甲方直观看到这不是几张孤立页面，而是一套已经具备角色分层和后续扩展能力的校园平台壳体。

## 2. 演示入口
### 管理端登录
- 路径：`apps/admin-web`
- 账号 1：`admin / ant.design`
- 账号 2：`teacher / ant.design`

### 家长端入口
- 路径：`apps/parent-h5`
- 当前不做真实登录，直接进入家长端 H5 演示页

## 3. 推荐讲解顺序
1. 先打开管理端登录页。
2. 说明这是“同一套系统，不同角色入口不同”。
3. 登录总控端，展示总览驾驶舱。
4. 切到风险中心和系统配置。
5. 退出后用老师账号登录。
6. 展示老师工作台、我的班级、待办中心。
7. 最后打开家长端 H5。
8. 展示首页、消息、服务里的请假申请。

## 4. 每一步要讲什么
### 登录页
- 这里不是普通后台登录，而是直接把演示角色入口做出来。
- 当前演示角色是总控端和老师端，家长端单独用移动 H5 展示。

### 总控端首页
- 强调“全校视角”。
- 强调总控端看到的是统一事件故事线。
- 指出当前 demo 用“张小雨请假申请”串联三端。

### 风险中心
- 强调平台侧不是做算法细节，而是做统一事件治理和处置视图。
- 当前风险中心展示的是待审批、待回复、待复核这类统一待办。

### 系统配置
- 强调产品不是一次性定制页面。
- 不同学校后续可以通过角色模板、组织结构、配置项进行迁移。

### 老师工作台
- 强调老师看到的是班级日常，而不是缩小版后台。
- 待处理事项与家长端请假申请形成闭环。

### 待办中心
- 强调老师的工作入口稳定清晰。
- 说明未来接入 AI、识别、消息时，只是把能力挂进待办流，不会推翻页面结构。

### 家长端首页
- 强调家长端要轻量、可信、易懂。
- 重点展示“孩子状态”“通知触达”“服务入口”。

### 家长端消息与请假
- 说明家长提交请假后，老师端会看到待审批事项。
- 这证明三端不是孤立页面，而是同一业务故事线。

## 5. 最后一句总结
当前阶段先把平台壳体、角色分层和演示故事线做完整，后续视觉识别、设备接入、无人机联动等能力都可以在这套结构上继续挂接。
